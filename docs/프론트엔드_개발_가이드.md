# 프론트엔드 개발 가이드

## 프론트엔드 역할 요약

```
[웹캠] → [MediaPipe 손 추출] → [백엔드 AI 추론] → [자모 조합] → [화면 표시]
   ①            ②                    ③                ④             ⑤
```

| 단계 | 담당 | 설명 |
|------|------|------|
| ① 웹캠 영상 캡처 | 프론트 | 키오스크 브라우저에서 웹캠 접근 |
| ② 손 랜드마크 추출 | 프론트 | MediaPipe Hands로 좌표 63개 추출 |
| ③ AI 추론 | 백엔드 | 좌표 → 자모 1개 반환 (`POST /api/inference`) |
| ④ 자모 → 단어 조합 | 프론트 | 두벌식 오토마타로 한글 음절 조합 |
| ⑤ 화면 표시 + 통신 | 프론트 | 조합 결과 표시 + 역무원 답변 폴링 |

## 1단계: 웹캠 + MediaPipe 손 랜드마크 추출

### MediaPipe Hands (JavaScript)

브라우저에서 MediaPipe Hands를 실행하여 손 랜드마크 21개(x, y, z)를 추출합니다.

**설치:**

```html
<script src="https://cdn.jsdelivr.net/npm/@mediapipe/hands/hands.js" crossorigin="anonymous"></script>
<script src="https://cdn.jsdelivr.net/npm/@mediapipe/camera_utils/camera_utils.js" crossorigin="anonymous"></script>
```

또는 npm:

```bash
npm install @mediapipe/hands @mediapipe/camera_utils
```

**기본 사용 예시:**

```javascript
const hands = new Hands({
  locateFile: (file) => `https://cdn.jsdelivr.net/npm/@mediapipe/hands/${file}`
});

hands.setOptions({
  maxNumHands: 1,
  modelComplexity: 1,
  minDetectionConfidence: 0.5,
  minTrackingConfidence: 0.5,
});

hands.onResults((results) => {
  if (results.multiHandLandmarks && results.multiHandLandmarks.length > 0) {
    const landmarks = results.multiHandLandmarks[0]; // 21개 랜드마크
    // landmarks[i].x, landmarks[i].y, landmarks[i].z
    
    // 63개 float 배열로 변환
    const keypoints = [];
    for (const lm of landmarks) {
      keypoints.push(lm.x, lm.y, lm.z);
    }
    
    // 백엔드로 전송
    sendToBackend(keypoints);
  }
});
```

### 출력 데이터 형식

```
keypoints = [x0, y0, z0, x1, y1, z1, ..., x20, y20, z20]
```

- 총 63개 float (21개 랜드마크 x 3좌표)
- 각 값은 0.0 ~ 1.0 범위 (정규화된 좌표)

## 2단계: 백엔드 통신 (API 4개)

### API 1. 좌표 → AI 추론 요청

```javascript
// 키오스크 → 서버: 좌표를 보내고 인식된 자모를 받음
async function sendToBackend(keypoints) {
  const res = await fetch("/api/inference", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      device_id: "SEOUL_01",
      keypoints: keypoints    // float 63개 배열
    })
  });
  const data = await res.json();
  // data = { log_id: 10, recognized_word: "ㅏ", confidence: 0.95 }
  return data;
}
```

**응답:**

```json
{
  "log_id": 10,
  "recognized_word": "ㅏ",
  "confidence": 0.95
}
```

## 3단계: 한글 자모 조합기 (JS 포팅 필요)

백엔드가 반환하는 자모(`ㅅ`, `ㅏ`, `ㄱ`, `ㅗ`, `ㅏ`)를 받아 실시간으로 한글 음절(`사과`)을 조합합니다.

### 3-5. 음절 조합/분해 함수

```javascript
function compose(cho, jung, jong = '') {
  return String.fromCharCode(
    0xAC00 + (CHOSUNG.indexOf(cho) * 21 + JUNGSUNG.indexOf(jung)) * 28 + JONGSUNG.indexOf(jong)
  );
}

function decompose(syllable) {
  const code = syllable.charCodeAt(0) - 0xAC00;
  const jongIdx = code % 28;
  const jungIdx = Math.floor(code / 28) % 21;
  const choIdx = Math.floor(code / 28 / 21);
  return [CHOSUNG[choIdx], JUNGSUNG[jungIdx], JONGSUNG[jongIdx]];
}
```

### 3-6. 핵심 조합 로직

자모 입력 시 아래 규칙을 순서대로 적용합니다.

**자음 입력 시 (`_inputConsonant`):**

```
1. 현재 중성(모음) 없음:
   a. 현재 초성과 같은 자음이고 쌍자음 가능 → 쌍자음으로 합침
      예: 초성 ㄱ + ㄱ → ㄲ
   b. 이미 초성 있으면 → 기존 초성 확정, 새 초성 시작
   c. 초성 없으면 → 초성으로 설정

2. 현재 중성(모음) 있음 (종성 자리):
   a. 종성 없음 → 종성으로 설정
   b. 종성과 같은 자음이고 쌍자음 가능 → 종성 쌍자음
      예: 종성 ㅅ + ㅅ → ㅆ (있다 → 있+ㅆ)
   c. 복합 종성 가능 → 합침 (ㄹ+ㄱ → ㄺ 등)
   d. 그 외 → 현재 음절 확정, 새 초성 시작
```

**모음 입력 시 (`_inputVowel`):**

```
1. 초성 없음 → ㅇ(묵음) + 모음
   예: ㅏ → 아

2. 초성만 있음 → 중성 추가
   예: ㄱ + ㅏ → 가

3. 초성+중성, 종성 없음 → 복합 모음 시도
   a. 가능하면 합침 (ㅗ+ㅏ → ㅘ)
   b. 불가하면 현재 음절 확정, 새 ㅇ+모음

4. 종성 있음 → 종성이 다음 초성으로 이동 ⭐ (가장 중요)
   a. 복합 종성 → 첫째는 종성 유지, 둘째가 다음 초성
      예: 닭+ㅏ → 달가 (ㄺ → ㄹ 남기고 ㄱ 이동)
   b. 단순 종성 → 종성 빠지고 다음 초성으로
      예: 한+ㅡ → 하느 (ㄴ이 다음 초성으로 이동)
```

## 4단계: Dwell 타이머 (입력 확정 로직)

지문자 인식은 매 프레임 결과가 나오므로, 같은 자모를 **일정 시간 유지**해야 확정되는 Dwell 방식을 사용합니다.

Python 원본 파일 `word_composer/word_builder.py`를 참고합니다.

### 타이머 설정값

| 항목 | 값 | 설명 |
|------|-----|------|
| `DWELL_SECS` | 1.0초 | 같은 자모를 1초 유지하면 확정 |
| `SPACE_SECS` | 2.0초 | 손 없음(none) 2초 유지하면 공백 추가 |
| `COOLDOWN` | 0.5초 | 확정 직후 동일 자모 재입력 방지 |

### 동작 방식

```
매 프레임:
  1. 백엔드에서 인식 결과(자모) 수신
  2. confidence < 0.75 이면 무시
  3. 이전 프레임과 같은 자모인지 확인:
     - 같으면 → 유지 시간 누적
     - 다르면 → 타이머 리셋
  4. 유지 시간 >= DWELL_SECS 이면:
     - 자모 확정 → 조합기(KoreanComposer)에 전달
     - 쿨다운 시작
  5. 손 인식 안 됨(none) 유지 시간 >= SPACE_SECS 이면:
     - 현재 조합 확정 + 공백 추가
```

### JS 구현 예시

```javascript
class WordBuilder {
  constructor(dwellSecs = 1.0, spaceSecs = 2.0, cooldown = 0.5) {
    this.dwellSecs = dwellSecs;
    this.spaceSecs = spaceSecs;
    this.cooldown = cooldown;
    this.composer = new KoreanComposer();

    this._curLabel = null;
    this._labelStart = 0;
    this._lastCommitted = null;
    this._committedAt = 0;
  }

  update(label) {
    const now = performance.now() / 1000;
    label = (label && label !== 'none') ? label : null;

    if (label !== this._curLabel) {
      this._curLabel = label;
      this._labelStart = now;
      this._lastCommitted = null;
    }

    const elapsed = now - this._labelStart;
    const canCommit = (now - this._committedAt) > this.cooldown;
    let committed = null;

    if (label === null) {
      const progress = Math.min(elapsed / this.spaceSecs, 1.0);
      if (elapsed >= this.spaceSecs && canCommit && this.composer.composing) {
        this.composer.space();
        this._committedAt = now;
        this._lastCommitted = null;
        committed = ' ';
      }
      return { text: this.composer.text, composing: this.composer.composing, progress, committed };
    } else {
      const progress = Math.min(elapsed / this.dwellSecs, 1.0);
      if (elapsed >= this.dwellSecs && canCommit && label !== this._lastCommitted) {
        this.composer.add(label);
        this._lastCommitted = label;
        this._committedAt = now;
        committed = label;
      }
      return { text: this.composer.text, composing: this.composer.composing, progress, committed };
    }
  }

  backspace() {
    this.composer.backspace();
    this._lastCommitted = null;
  }

  clear() {
    this.composer.clear();
    this._lastCommitted = null;
    this._curLabel = null;
  }
}
```

## 주의사항

| 항목 | 내용 |
|------|------|
| MediaPipe 버전 | `@mediapipe/hands` 최신 버전 사용 (JS용은 solutions API 정상 지원) |
| 추론 호출 빈도 | 매 프레임 호출하면 서버 부하 큼 → **3~5 프레임마다 1회** 호출 권장 |

| CORS | 백엔드에서 키오스크 도메인 CORS 허용 필요 |
