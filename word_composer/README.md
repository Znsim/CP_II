# 지문자 → 단어 조합기 설계 문서

## 1. 왜 표준 입력기 방식인가?

지문자(수어)는 **자모 단위**로 입력된다.  
예) 사과 → ㅅ → ㅏ → ㄱ → ㅗ → ㅏ (총 5번 제스처)

이 자모들을 읽기 쉬운 완성형 한글로 바꾸려면,  
PC 한글 키보드와 동일한 **두벌식 오토마타(state machine)** 를 사용한다.

---

## 2. 표준 두벌식 입력기 원리

### 2-1. 한글 음절 구조

모든 한글 음절은 세 요소로 구성된다.

```
음절 = 초성(자음) + 중성(모음) + 종성(자음, 생략 가능)
예)  닭  =   ㄷ   +   ㅏ   +   ㄺ(ㄹ+ㄱ)
     가  =   ㄱ   +   ㅏ   +   없음
```

완성형 유니코드 계산식:

```
코드포인트 = 0xAC00 + (초성_인덱스 × 21 + 중성_인덱스) × 28 + 종성_인덱스
```

| 구분 | 개수 | 목록 |
|------|------|------|
| 초성 | 19개 | ㄱㄲㄴㄷㄸㄹㅁㅂㅃㅅㅆㅇㅈㅉㅊㅋㅌㅍㅎ |
| 중성 | 21개 | ㅏㅐㅑㅒㅓㅔㅕㅖㅗㅘㅙㅚㅛㅜㅝㅞㅟㅠㅡㅢㅣ |
| 종성 | 28개 | (없음) ㄱㄲㄳㄴㄵㄶㄷㄹㄺㄻㄼㄽㄾㄿㅀㅁㅂㅄㅅㅆㅇㅈㅊㅋㅌㅍㅎ |

---

### 2-2. 상태 머신 (State Machine)

입력기는 4가지 상태를 순환한다.

```
[EMPTY] ──자음──▶ [CHO] ──모음──▶ [CHO+JUNG] ──자음──▶ [CHO+JUNG+JONG]
   ▲                 │                  │                      │
   │                 │                  │                      │
   └──────────────자음(확정)────────────┴──────────────────────┘
                                                              모음 → 종성이 다음 초성으로 이동
```

#### 상태 전이 규칙 (핵심)

| 현재 상태 | 입력 | 동작 |
|-----------|------|------|
| EMPTY | 자음 | 초성으로 설정 → CHO |
| EMPTY | 모음 | ㅇ을 초성으로, 모음을 중성으로 → CHO+JUNG |
| CHO | 자음 | 이전 초성 확정, 새 초성 시작 → CHO |
| CHO | 모음 | 중성 추가 → CHO+JUNG |
| CHO+JUNG | 자음 | 종성 후보로 설정 → CHO+JUNG+JONG |
| CHO+JUNG | 모음 | 복합모음 가능하면 합침, 아니면 현재 확정 후 새 음절 |
| CHO+JUNG+JONG | **모음** | ⭐ 종성이 다음 초성으로 이동 (핵심 규칙) |
| CHO+JUNG+JONG | 자음 | 복합종성 가능하면 합침, 아니면 현재 확정 후 새 초성 |

---

### 2-3. 핵심 규칙: 받침의 모호성 해결 ⭐

```
입력 순서: ㅅ → ㅏ → ㄱ → ㅘ(ㅗ+ㅏ)
                             ↑
                  ㄱ는 '삭'의 받침인가? '과'의 초성인가?
```

**두벌식 규칙:** 종성(받침) 다음에 **모음이 오면**, 종성이 다음 음절의 초성으로 이동

```
단계별 조합 과정:
  ㅅ      →  ㅅ (초성 대기)
  ㅅ + ㅏ  →  사 (초성+중성)
  사 + ㄱ  →  삭 (종성 후보 - 아직 미확정)
  삭 + ㅗ  →  사 + ㄱ (ㄱ이 다음 초성으로 이동)
  사ㄱ + ㅏ →  사 + 가 (ㅗ+ㅏ → ㅘ는 복합모음으로 조합)

  최종: 사과 ✅
```

---

### 2-4. 복합 모음 (겹모음)

두 모음 입력을 하나의 모음으로 합친다.

| 입력 조합 | 결과 | 비고 |
|-----------|------|------|
| ㅗ + ㅏ | ㅘ | 과, 봐 |
| ㅗ + ㅐ | ㅙ | |
| ㅗ + ㅣ | ㅚ | ※ 데이터셋에 ㅚ 제스처 있음 (직접 입력 가능) |
| ㅜ + ㅓ | ㅝ | 원, 뭐 |
| ㅜ + ㅔ | ㅞ | |
| ㅜ + ㅣ | ㅟ | ※ 데이터셋에 ㅟ 제스처 있음 |
| ㅡ + ㅣ | ㅢ | ※ 데이터셋에 ㅢ 제스처 있음 |

> **이 프로젝트:** ㅚ, ㅟ, ㅢ는 제스처로 직접 입력.  
> ㅘ, ㅙ, ㅝ, ㅞ는 두 모음 조합으로만 입력 가능.

---

### 2-5. 복합 종성 (겹받침)

두 자음 입력을 하나의 받침으로 합친다.

| 입력 조합 | 결과 | 예시 단어 |
|-----------|------|-----------|
| ㄱ + ㅅ | ㄳ | 넋 |
| ㄴ + ㅈ | ㄵ | 앉다 |
| ㄴ + ㅎ | ㄶ | 많다 |
| ㄹ + ㄱ | ㄺ | 닭 |
| ㄹ + ㅁ | ㄻ | 삶 |
| ㄹ + ㅂ | ㄼ | 밟다 |
| ㄹ + ㅅ | ㄽ | 옳다 |
| ㄹ + ㅌ | ㄾ | |
| ㄹ + ㅍ | ㄿ | |
| ㄹ + ㅎ | ㅀ | 싫다 |
| ㅂ + ㅅ | ㅄ | 없다 |

## 3. Dwell(유지) 입력 방식

웹캠에서 자모를 입력받을 때, 매 프레임마다 인식 결과가 바뀌면 오작동한다.  
**같은 자모를 일정 시간(1초) 유지했을 때만 입력으로 확정**한다.

```
시간 →
[ㅅ 인식 시작]━━━━━[1.0초 경과 → ㅅ 확정!]━━[쿨다운 0.5초]━━━━...
                                             (이 사이에 같은 글자 재입력 방지)

[none/손없음]━━━━━━[2.0초 경과 → 현재 음절 확정 + 공백]
```

### 파라미터 (조정 가능)

| 파라미터 | 기본값 | 의미 |
|----------|--------|------|
| `dwell` | 1.0초 | 자모 확정에 필요한 유지 시간 |
| `space_dwell` | 2.0초 | none 유지 시 공백 입력 |
| `cooldown` | 0.5초 | 확정 후 동일 자모 재입력 방지 시간 |

## 4. 파일 구조

```
word_composer/
├── README.md           ← 이 문서
├── korean_composer.py  ← 한글 자모 조합 엔진 (KoreanComposer)
└── word_builder.py     ← Dwell 입력 + 조합 통합 (WordBuilder)
```

### 사용 방법 (realtime_spelling_demo.py에서)

```python
from word_composer.word_builder import WordBuilder

builder = WordBuilder(dwell=1.0, space_dwell=2.0)

# 매 프레임
result = builder.update(label)   # label = 현재 인식된 자모 or None
print(result['text'])            # "사과"
print(result['composing'])       # 현재 조합 중인 음절 (미확정)
print(result['progress'])        # 0.0 ~ 1.0 (dwell 진행도)

# 키보드 조작
builder.backspace()              # 마지막 입력 취소
builder.clear()                  # 전체 초기화
```

## 5. 조합 예시
